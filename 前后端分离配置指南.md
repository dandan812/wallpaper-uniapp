# 前后端分离配置指南

## 概述

本项目采用前后端分离架构:
- **前端**: uni-app (Vue3) 小程序
- **后端**: Node.js + Express API 服务

## 前端配置

### 1. 环境配置

前端配置文件位于 `config/env.js`:

```javascript
const ENV = {
  // 开发环境
  development: {
    baseURL: 'http://localhost:3000/api',
    accessKey: 'key123456'
  },
  // 生产环境
  production: {
    baseURL: 'https://api.yourdomain.com/api',
    accessKey: 'your_production_key'
  }
};
```

### 2. 配置步骤

1. 复制配置示例文件:
```bash
cp config/env.example.js config/env.js
```

2. 修改 `config/env.js` 中的配置:
   - `baseURL`: 后端 API 地址
   - `accessKey`: API 访问密钥

### 3. 开发环境配置

**本地开发**:
```javascript
development: {
  baseURL: 'http://localhost:3000/api',
  accessKey: 'key123456'
}
```

**局域网测试** (手机访问电脑):
```javascript
development: {
  baseURL: 'http://192.168.1.100:3000/api',  // 替换为你的电脑 IP
  accessKey: 'key123456'
}
```

### 4. 生产环境配置

```javascript
production: {
  baseURL: 'https://api.yourdomain.com/api',
  accessKey: 'your_production_key'
}
```

### 5. 动态 Access Key

支持通过 URL 参数传递 accessKey:

```javascript
// 页面跳转时传递
uni.navigateTo({
  url: '/pages/index/index?accessKey=custom_key'
});
```

## 后端配置

### 1. 环境变量

后端配置文件 `.env`:

```env
NODE_ENV=production
PORT=3000

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=wallpaper
DB_USER=wallpaper_user
DB_PASSWORD=your_password

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password

# API密钥 (多个用逗号分隔)
ACCESS_KEYS=key123456,key789012,your_production_key
```

### 2. CORS 配置

后端已配置 CORS 支持跨域请求:

```javascript
app.use(cors());
```

如需限制特定域名:

```javascript
app.use(cors({
  origin: ['https://yourdomain.com', 'http://localhost:8080'],
  credentials: true
}));
```

### 3. Nginx 配置

如果使用 Nginx 反向代理:

```nginx
server {
    listen 80;
    server_name api.yourdomain.com;

    # CORS 配置
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';
    add_header Access-Control-Allow-Headers 'access-key, Content-Type';

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## API 接口说明

### 请求格式

所有请求需要在请求头中携带 `access-key`:

```javascript
header: {
  'access-key': 'key123456'
}
```

### 响应格式

统一的响应格式:

```json
{
  "errCode": 0,
  "errMsg": "查询成功",
  "data": {}
}
```

- `errCode`: 0=成功, 非0=失败
- `errMsg`: 提示信息
- `data`: 返回数据

### 错误码

- `0`: 成功
- `400`: 参数错误
- `401`: 未授权 (access-key 错误)
- `404`: 资源不存在
- `500`: 服务器错误

## 接口列表

### 分类相关
- `GET /api/classify` - 分类列表
- `GET /api/classify?select=1` - 推荐分类

### 轮播图
- `GET /api/homeBanner` - 轮播图列表

### 壁纸相关
- `GET /api/wallList?classid=1&limit=10&skip=0` - 壁纸列表

- `GET /api/randomWall?limit=9` - 随机壁纸

### 公告相关
- `GET /api/wallNewsList` - 公告列表
- `GET /api/wallNewsDetail/:id` - 公告详情

### 用户相关
- `GET /api/userInfo?userId=1` - 用户信息
- `POST /api/setupScore` - 评分
- `POST /api/downloadWall` - 下载记录
- `GET /api/userWallList?userId=1&type=score` - 用户历史

### 3. 常见问题

**问题1: 跨域错误**
```
Access to XMLHttpRequest at 'http://localhost:3000/api/classify'
from origin 'http://localhost:8080' has been blocked by CORS policy
```

解决: 确保后端已配置 CORS

**问题2: access-key 错误**
```json
{
  "errCode": 401,
  "errMsg": "无效的 access-key"
}
```

解决: 检查前端 `config/env.js` 中的 accessKey 是否与后端 `.env` 中的 ACCESS_KEYS 匹配

**问题3: 接口 404**
```json
{
  "errCode": 404,
  "errMsg": "Not Found"
}
```

解决: 检查 API 路径是否正确,参考 `api/apis.js`

## 部署流程

### 1. 前端部署

**小程序**:
1. 修改 `config/env.js` 为生产环境配置
2. 使用 HBuilderX 发行到微信小程序
3. 上传到微信公众平台
4. 提交审核

**H5**:
1. 修改 `config/env.js` 为生产环境配置
2. 使用 HBuilderX 发行到 H5
3. 上传到服务器

### 2. 后端部署

参考 [backend-code/QUICK_START.md](../backend-code/QUICK_START.md)

### 3. 域名配置

1. 购买域名
2. 配置 DNS 解析:
   - `api.yourdomain.com` -> 服务器 IP
3. 配置 HTTPS 证书 (推荐使用 Let's Encrypt)
4. 更新前端配置中的 baseURL

## 安全建议

1. **生产环境使用 HTTPS**
2. **定期更换 access-key**
3. **限制 CORS 允许的域名**
4. **添加请求频率限制**
5. **敏感数据加密传输**
