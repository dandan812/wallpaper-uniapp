# 后端代码阅读说明

这份说明按当前统一后的接口命名来写，适合第一次系统看这个后端的人。

## 1. 先记住请求流向

一个请求大致按这条链路执行：

```text
前端请求 /api/categories
-> app.js
-> middlewares/accessKey.js
-> routes/category.js
-> controllers/categoryController.js
-> models/Category.js
-> MySQL / Redis
-> utils/response.js
```

读任何一个接口，都按这条线往下追。

## 6. 当前阅读时要注意的兼容点

- 返回里同时保留 `id` 和 `_id`
- 新参数主名：
  - `wallpaperId`
  - `score`
  - `limit`
  - `skip`
- 旧参数仍兼容：
  - `wallId`
  - `userScore`
- `userId` 不传时，部分接口会回退到默认用户
- [Wallpaper.js](C:/Users/hulian/Desktop/huliang/bizi/uniappvue3/backend-code/src/models/Wallpaper.js) 里有字段映射：
  - `smallPicurl -> small_picurl`

## 7. 一句话的阅读方法

不要一上来把整个后端全看完，先从一个接口开始，按：

`route -> controller -> model -> response`

往下追即可。
