# 前后端配置

这份文档只记录当前生效的前后端配置规则，默认按新主路径理解接口。

## 相关文件

- `config/env.js`：前端实际环境配置
- `api/apis.js`：前端接口定义
- `utils/request.js`：统一请求封装
- `../backend-code/.env.example`：后端环境变量模板
- `../backend-code/.env`：后端运行配置

## 前端配置方式

前端通过 `config/env.js` 提供两个核心值：

- `BASE_URL`：接口基础地址
- `ACCESS_KEY`：请求头里的 `access-key`

请求流程：

1. `api/apis.js` 定义接口路径
2. `utils/request.js` 将路径拼到 `BASE_URL`
3. `getAccessKey()` 生成请求头 `access-key`

## 当前接口主路径

前端主调用统一使用：

- `/banners`
- `/categories`
- `/notices`
- `/notices/:id`
- `/wallpapers`
- `/wallpapers/:id`
- `/wallpapers/random`
- `/wallpapers/search`
- `/users/me`
- `/users/wallpapers`
- `POST /wallpapers/score`
- `POST /wallpapers/download`

后端保留旧路径兼容，但文档和前端都不再以旧路径为主。

## 常见环境示例

### 本机 H5 联调

```javascript
development: {
  baseURL: "http://localhost:3000/api",
  accessKey: "key123456"
}
```

### 小程序局域网联调

```javascript
development: {
  baseURL: "http://192.168.1.100:3000/api",
  accessKey: "key123456"
}
```

### 生产环境

```javascript
production: {
  baseURL: "http://8.135.46.112/api",
  accessKey: "key123456"
}
```

## 后端环境变量

`../backend-code/.env` 至少需要：

```env
NODE_ENV=production
PORT=3000

DB_HOST=localhost
DB_PORT=3306
DB_NAME=wallpaper
DB_USER=wallpaper_user
DB_PASSWORD=your_mysql_password

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password

ACCESS_KEYS=key123456,key789012
```

说明：

- `ACCESS_KEYS` 支持多个值，使用英文逗号分隔
- 生产环境不要把真实密码写进仓库

## 联调检查清单

- `config/env.js` 的 `baseURL` 是否指向当前 API
- `config/env.js` 的 `accessKey` 是否在后端 `ACCESS_KEYS` 中
- 浏览器或小程序请求头里是否真的带了 `access-key`
- 后端是否已监听 `PORT`
- 当前请求是否命中新主路径，而不是旧兼容别名
