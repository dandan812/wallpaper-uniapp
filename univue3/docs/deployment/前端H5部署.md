# 前端H5部署

这份文档以当前线上部署方式为准，适用于把 H5 页面发布到阿里云服务器 `8.135.46.112`。

当前线上目录：

- 前端目录：`/var/www/wallpaper-h5`
- 访问地址：[http://8.135.46.112](http://8.135.46.112)

## 1. 发布前先确认生产环境

先检查 [env.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/config/env.js) 的生产配置：

## 3. 推荐部署方式

推荐使用项目根目录的脚本：

- [deploy-h5-local.sh](C:/Users/hulian/Desktop/huliang/bizi/univue3/deploy-h5-local.sh)

这个脚本当前会自动执行：

1. 清空服务器旧前端目录 `/var/www/wallpaper-h5`
2. 上传新的 `unpackage/dist/build/web/*`
3. 修正服务器文件权限

权限修正规则：

- 目录：`755`
- 文件：`644`
- 属主：`www-data:www-data`

这样可以避免：

- 旧哈希资源残留
- 浏览器继续命中旧 `css/js`
- Nginx 因权限问题返回 `403`

## 4. 脚本执行方式

如果你在 Git Bash 里操作：

```bash
chmod +x deploy-h5-local.sh
./deploy-h5-local.sh
```

如果你在 PowerShell 里操作，更推荐直接手动执行这两步：

```powershell
ssh root@8.135.46.112 "mkdir -p /var/www/wallpaper-h5 && find /var/www/wallpaper-h5 -mindepth 1 -maxdepth 1 -exec rm -rf {} +"
scp -r .\unpackage\dist\build\web\* root@8.135.46.112:/var/www/wallpaper-h5/
```

上传后还需要修权限：

```powershell
ssh root@8.135.46.112 "chown -R www-data:www-data /var/www/wallpaper-h5 && find /var/www/wallpaper-h5 -type d -exec chmod 755 {} + && find /var/www/wallpaper-h5 -type f -exec chmod 644 {} +"
```

## 6. 为什么有时“页面没变化”

当前 Nginx 对静态资源启用了强缓存。

所以发布后如果你觉得页面没更新，优先这样处理：

- Windows 浏览器按 `Ctrl + F5`
- 手机浏览器清缓存后重开
- 或直接用无痕模式打开

说明：

- 每次重新发行后，通常会生成新的带哈希文件名的资源
- `index.html` 会自动引用新的文件名
- 只要新包上传成功，强刷后就能看到最新页面

## 7. 常见问题

### 页面空白或手机打不开

先检查静态资源是否能正常访问：

- `http://8.135.46.112/assets/index-*.js`
- `http://8.135.46.112/assets/index-*.css`

如果返回 `403`，通常就是服务器目录权限不对，重新执行权限修复命令即可。

### 页面还是旧版本

- 确认你改的是源码，不是旧构建产物
- 确认已经重新发行 H5
- 确认已经上传最新 `unpackage/dist/build/web/*`
- 最后做一次强刷

### 资源 404

- 检查服务器目录 `/var/www/wallpaper-h5` 是否已上传成功
- 检查 `index.html` 是否引用了新哈希文件
- 检查 Nginx `root` 是否仍指向 `/var/www/wallpaper-h5`

### API 请求失败

- 检查 [env.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/config/env.js) 的生产地址
- 检查 Nginx `/api` 反代
- 检查 PM2 中 `wallpaper-api` 是否在线
