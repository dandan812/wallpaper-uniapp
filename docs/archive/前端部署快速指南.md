# 前端部署到服务器 - 快速指南

> 归档说明：这是旧版快速指南，当前前端 H5 部署请优先参考 `docs/deployment/前端H5部署.md`。

## 服务器信息
- **IP**: 8.135.46.112
- **系统**: Ubuntu 24.04
- **配置**: 2核2G

## 部署步骤

### 方式一：自动化部署（推荐）

#### 第一步：在服务器上配置环境

```bash
# 1. SSH 登录服务器
ssh root@8.135.46.112

# 2. 下载并执行服务器配置脚本
cd /root
# 上传 deploy-h5-server.sh 到服务器
chmod +x deploy-h5-server.sh
./deploy-h5-server.sh
```

#### 第二步：在本地打包并上传

```bash
# 1. 使用 HBuilderX 发行到 H5
#    菜单 → 发行 → 网站-H5移动版

# 2. 执行上传脚本
chmod +x deploy-h5-local.sh
./deploy-h5-local.sh
```

### 方式二：手动部署

#### 1. 配置前端环境变量

编辑 `config/env.js`:

```javascript
production: {
  baseURL: 'http://8.135.46.112/api',  // 使用 Nginx 代理
  accessKey: 'key123456'
}
```

#### 2. 使用 HBuilderX 编译

- 打开 HBuilderX
- 点击菜单: **发行 → 网站-H5移动版**
- 等待编译完成

编译后的文件在: `unpackage/dist/build/h5/`

#### 3. 上传到服务器

**使用 SCP**:
```bash
cd unpackage/dist/build/h5
scp -r * root@8.135.46.112:/var/www/wallpaper-h5/
```

**或使用 WinSCP/FileZilla**:
- 连接到 8.135.46.112
- 上传到 `/var/www/wallpaper-h5/`

#### 4. 在服务器上配置 Nginx

```bash
# SSH 登录
ssh root@8.135.46.112

# 创建配置文件
nano /etc/nginx/sites-available/wallpaper-h5
```

粘贴以下内容:

```nginx
server {
    listen 80;
    server_name 8.135.46.112;

    root /var/www/wallpaper-h5;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
}
```

启用配置:
```bash
ln -s /etc/nginx/sites-available/wallpaper-h5 /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

#### 5. 测试访问

浏览器打开: http://8.135.46.112

## 常见问题

### 1. 页面空白
- 检查浏览器控制台错误
- 确认文件已正确上传
- 检查 Nginx root 路径

### 2. API 请求失败
```bash
# 检查后端是否运行
pm2 status

# 检查端口
netstat -tulpn | grep 3000
```

### 3. 刷新页面 404
- 确认 Nginx 配置了 `try_files $uri $uri/ /index.html;`

### 4. 静态资源 404
- 检查文件路径是否正确
- 查看 Nginx 错误日志: `tail -f /var/log/nginx/error.log`
