## 📋 配置步骤

### 前端配置

1. **复制配置文件**:
```bash
cp config/env.example.js config/env.js
```

2. **修改配置**:
编辑 `config/env.js`,设置:
- 开发环境 API 地址
- 生产环境 API 地址
- Access Key

3. **使用配置向导** (可选):
```bash
bash setup-config.sh
```

### 后端配置

1. **安装依赖**:
```bash
cd backend-code
npm install
```

2. **配置环境变量**:
```bash
cp .env.example .env
# 编辑 .env 文件
```

3. **启动服务**:
```bash
# 开发环境
npm run dev

# 生产环境
pm2 start ecosystem.config.js
```

## 🚀 快速开始

### 本地开发

1. **启动后端**:
```bash
cd backend-code
npm install
npm run dev
```

2. **配置前端**:
```bash
# 编辑 config/env.js
development: {
  baseURL: 'http://localhost:3000/api',
  accessKey: 'key123456'
}
```

3. **运行前端**:
使用 HBuilderX 运行到浏览器或小程序

### 生产部署

1. **部署后端**:
参考 [快速开始](../getting-started/快速开始.md)

2. **配置前端**:
```bash
# 编辑 config/env.js
production: {
  baseURL: 'https://api.yourdomain.com/api',
  accessKey: 'your_production_key'
}
```

3. **发布前端**:
使用 HBuilderX 发行到小程序或 H5

## 🔐 安全配置

### Access Key 管理

**后端配置** (.env):
```env
ACCESS_KEYS=key123456,key789012,production_key
```

**前端配置** (config/env.js):
```javascript
accessKey: 'key123456'
```

### HTTPS 配置

生产环境建议使用 HTTPS:

1. 申请 SSL 证书 (Let's Encrypt)
2. 配置 Nginx HTTPS
3. 更新前端 baseURL 为 https://

## 📊 测试验证

### 1. 测试后端

### 2. 测试前端

1. 运行前端项目
2. 打开浏览器开发者工具
3. 查看 Network 标签
4. 检查请求是否正常

### 3. 常见问题

**跨域错误**:
- 确保后端已配置 CORS
- 检查 Nginx 配置

**Access Key 错误**:
- 检查前后端 Access Key 是否匹配
- 查看请求头是否正确携带

**接口 404**:
- 检查 API 路径是否正确
- 查看后端路由配置
