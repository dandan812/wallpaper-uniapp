# 快速开始

这份文档的目标是让你在最短时间内跑通当前项目。项目由 uni-app 前端和 `backend-code/` 下的 Node.js API 组成，前端通常通过 HBuilderX 运行，后端通过 Node.js 启动。

## 1. 环境准备

- HBuilderX
- Node.js 18+
- MySQL 8+
- Redis 6+

## 2. 项目结构速览

```text
univue3/
├── api/                  # 前端接口定义
├── config/               # 前端环境配置
├── utils/                # 前端请求封装
├── pages/                # uni-app 页面
├── backend-code/         # 后端 API 项目
└── docs/                 # 项目文档
```

## 3. 配置前端

1. 复制配置文件。

```bash
cp config/env.example.js config/env.js
```

2. 按实际环境修改 `config/env.js`。

开发环境最常见配置：

```javascript
development: {
  baseURL: 'http://localhost:3000/api',
  accessKey: 'key123456'
}
```

如果你不想手动改，也可以执行：

```bash
bash setup-config.sh
```

## 4. 启动后端

1. 安装依赖。

```bash
cd backend-code
npm install
```

2. 复制环境变量模板。

```bash
cp .env.example .env
```

3. 根据本机数据库和 Redis 实际信息修改 `.env`。

4. 初始化用户相关表。

```bash
mysql -u wallpaper_user -p wallpaper < init-user-tables.sql
```

5. 启动 API。

```bash
npm run dev
```

默认启动地址：

- 根检查接口：`http://localhost:3000/`
- API 前缀：`http://localhost:3000/api`

## 5. 启动前端

- 用 HBuilderX 打开项目根目录 `univue3`
- 运行到浏览器、微信开发者工具或其他目标平台

## 6. 最小验证

后端启动后，先验证健康检查：

```bash
curl http://localhost:3000/
```

预期返回：

```json
{"message":"壁纸 API 服务运行中"}
```

再验证一个需要 `access-key` 的接口：

```bash
curl -H "access-key: key123456" "http://localhost:3000/api/classify"
```

## 7. 常用命令

```bash
# 启动后端开发服务
cd backend-code && npm run dev

# 生产方式运行后端
cd backend-code && pm2 start ecosystem.config.js
```
