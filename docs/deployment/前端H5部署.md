# 前端 H5 部署

项目提供了两种 H5 部署方式：手动上传构建产物，或使用仓库里的脚本辅助清理和上传。

## 1. 先确认生产环境地址

发布前先检查 [env.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/config/env.js) 的生产配置：

```javascript
production: {
  baseURL: 'http://8.135.46.112/api',
  accessKey: 'key123456'
}
```

如果后面换成域名，把 `baseURL` 改成实际域名即可。

## 2. 生成 H5 构建产物

使用 HBuilderX：

1. 打开项目 [univue3](C:/Users/hulian/Desktop/huliang/bizi/univue3)
2. 选择“发行 -> 网站-H5移动版”
3. 等待构建完成

构建输出目录：

```text
unpackage/dist/build/web/
```

## 3. 推荐方式：使用本地上传脚本

[deploy-h5-local.sh](C:/Users/hulian/Desktop/huliang/bizi/univue3/deploy-h5-local.sh) 现在会执行两步：

- 先清空服务器目录 `/var/www/wallpaper-h5`
- 再上传最新的 `unpackage/dist/build/web/*`

执行方式：

```bash
chmod +x deploy-h5-local.sh
./deploy-h5-local.sh
```

这样做的好处是：

- 不会残留旧的哈希资源文件
- 避免浏览器继续加载旧版本 `css/js`
- 线上目录始终保持干净

## 4. 手动部署方式

如果你想手动操作，推荐按下面顺序执行：

```bash
ssh root@8.135.46.112 "mkdir -p /var/www/wallpaper-h5 && find /var/www/wallpaper-h5 -mindepth 1 -maxdepth 1 -exec rm -rf {} +"
scp -r unpackage/dist/build/web/* root@8.135.46.112:/var/www/wallpaper-h5/
```

## 5. 服务器目录

当前线上前端目录：

```text
/var/www/wallpaper-h5
```

Nginx 会直接从这个目录提供静态文件。

## 6. 刷新说明

线上静态资源开启了缓存。如果你上传后页面看起来没变化：

- Windows 浏览器按 `Ctrl + F5`
- 或打开无痕窗口重新访问
- 或清浏览器缓存

如果本次是重新打包后的新文件，通常 `index.html` 会自动引用新的哈希文件名，刷新后就会看到最新版本。

## 7. 验证清单

- [http://8.135.46.112](http://8.135.46.112) 可以正常打开
- 浏览器 Network 里 `css/js` 返回 `200`
- `/api/` 请求正常
- 页面刷新后仍然可访问

## 8. 常见问题

### 页面没变化

- 确认你改的是源码，不是旧的构建产物
- 确认已经重新执行 H5 发行
- 确认已经把 `unpackage/dist/build/web/*` 上传到服务器
- 上传后做一次强制刷新

### 资源 404

- 检查服务器目录是否真的有最新文件
- 检查 Nginx `root` 是否指向 `/var/www/wallpaper-h5`

### API 请求失败

- 检查 [env.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/config/env.js) 的生产地址
- 检查后端和 Nginx 反代是否正常
