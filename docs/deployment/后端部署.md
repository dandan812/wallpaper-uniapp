# 后端部署

后端代码位于 `backend-code/`。下面的步骤以把这部分部署到 Linux 服务器为准，默认运行目录是 `/var/www/wallpaper-api`。

## 1. 上传后端代码

推荐只上传 `backend-code/` 目录内容，而不是把整个前端项目一起放到 API 目录里。

```bash
mkdir -p /var/www/wallpaper-api
cd /var/www/wallpaper-api
```

你可以用 Git、SCP 或其他方式把本地 `backend-code/` 的文件同步到这里。

## 2. 安装依赖

```bash
cd /var/www/wallpaper-api
npm install --production
```

## 3. 配置环境变量

复制 `.env.example` 并修改实际值：

```bash
cp .env.example .env
```

最少需要确认这些字段：

```env
NODE_ENV=production
PORT=3000
DB_HOST=localhost
DB_PORT=3306
DB_NAME=wallpaper
DB_USER=wallpaper_user
DB_PASSWORD=your_password
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password
ACCESS_KEYS=key123456,key789012
```

## 4. 初始化数据库

项目仓库内提供了初始化 SQL：

```bash
mysql -u wallpaper_user -p wallpaper < init-user-tables.sql
```

## 5. 启动服务

先创建日志目录：

```bash
mkdir -p logs
```

开发方式：

```bash
npm run dev
```

生产方式：

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

## 6. Nginx 反向代理

仓库里已经有一份 `nginx.conf`，当前行为是：

- `/api/` 代理到 `http://localhost:3000/api/`
- `/` 返回简单文本，用于确认服务是否在线

示例配置：

```nginx
server {
    listen 80;
    server_name _;

    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }

    location / {
        return 200 'Wallpaper API Server';
        add_header Content-Type text/plain;
    }
}
```

启用后记得测试：

```bash
nginx -t
systemctl reload nginx
```

## 7. 部署后验证

```bash
# 健康检查
curl http://localhost:3000/

# 走 Nginx 的接口检查
curl -H "access-key: key123456" http://localhost/api/classify

# 查看 PM2 状态
pm2 status
```

## 8. 常用维护命令

```bash
pm2 logs wallpaper-api
pm2 restart wallpaper-api
pm2 stop wallpaper-api
pm2 monit
```

## 9. 注意事项

- `ecosystem.config.js` 里日志输出到 `./logs/`，所以目录必须提前存在。
- `.env.example` 里的密码是示例值，部署前一定要替换。
- 当前接口实际实现请以 `backend-code/src/routes/` 为准，详见 [API 测试](../reference/API测试.md)。
