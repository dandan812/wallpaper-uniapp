# 后端代码阅读说明

这份文档按“一个请求是怎么走完整条链路的”来整理，适合第一次系统阅读 `backend-code/` 时配合源码一起看。

## 1. 先记住后端主链路

后端主流程是：

```text
请求进入
-> app.js
-> middlewares
-> routes
-> controllers
-> models
-> MySQL / Redis
-> response.js 统一返回
```

你可以把它理解成：

- `app.js` 负责把服务搭起来
- `middlewares/` 负责公共校验和日志
- `routes/` 负责“这个路径交给谁处理”
- `controllers/` 负责具体业务
- `models/` 负责和数据库表打交道
- `config/` 负责数据库和 Redis 连接
- `utils/response.js` 负责统一响应结构

## 3. 入口层看什么

[app.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/app.js) 是后端入口，重点看三件事：

- 加了哪些全局中间件
- `/api` 下挂了哪些路由
- 错误处理中间件是不是放在最后

当前这里做了：

- 开启 `cors`
- 解析 `json` 和表单
- 挂请求日志
- 对所有 `/api` 请求统一做 `access-key` 校验
- 注册分类、轮播图、壁纸、公告、用户路由

根路径 `/` 只是探活接口，不参与业务。

## 4. 中间件层看什么

最关键的是：

- [accessKey.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/middlewares/accessKey.js)
- [logger.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/middlewares/logger.js)
- [errorHandler.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/middlewares/errorHandler.js)

它们分别负责：

- `accessKey.js`
  - 校验请求头里的 `access-key`
  - 兼容旧方式：`req.query.accessKey`
- `logger.js`
  - 记录每次请求的路径、状态码和耗时
- `errorHandler.js`
  - 兜住未捕获异常，统一返回 JSON

理解完这一层后，你就知道为什么所有接口都长得比较统一。

## 8. 配置层看什么

配置层主要有两个文件：

- [database.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/config/database.js)
- [redis.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/config/redis.js)

它们分别负责：

- Sequelize 连接 MySQL
- ioredis 连接 Redis

这里要特别注意：

- `.env` 里数据库和 Redis 地址
- 线上是本机连接，不是外部数据库

## 9. 返回格式怎么看

[response.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/utils/response.js)

这是整个项目响应风格统一的关键。

当前它兼容两种写法：

1. 直接写回响应

```js
success(res, data, '查询成功')
```

2. 先生成对象，再手动 `res.json`

```js
const result = success(data, '查询成功', total)
res.json(result)
```

这也是为什么新旧 controller 可以混着跑，而不用一次性重构。

## 10. 当前后端最重要的兼容点

你读代码时，最好一直带着下面这几个点：

- 所有接口都需要 `access-key`
- 返回里保留 `id` 和 `_id`
- 新参数优先用 `wallpaperId`、`score`、`limit`、`skip`
- 旧参数 `wallId`、`userScore`、`pageNum`、`pageSize` 仍兼容
- `userInfo` 和 `userWallList` 未传 `userId` 时会回退到首个用户
- `/homeBanner`、`/wallNewsList`、`/wallNewsDetail/:id` 是兼容旧前端的历史路径

## 11. 最适合练手的阅读方式

如果你想最快读懂这套后端，建议直接顺一遍这 3 条线：

1. 分类列表
- 路径：`/api/classify`
- 看 `routes/category.js -> controllers/categoryController.js -> models/Category.js`

2. 壁纸详情
- 路径：`/api/detailWall/:id`
- 看 `routes/wallpaper.js -> controllers/wallpaperController.js -> models/Wallpaper.js`

3. 用户评分
- 路径：`/api/setupScore`
- 看 `routes/user.js -> controllers/userController.js -> models/Score.js`

这三条线看完，你基本就能理解整个项目的大部分后端模式。
