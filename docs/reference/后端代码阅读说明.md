# 后端代码阅读说明

这份说明是给“几乎没写过后端”的你准备的。

你不用一开始就理解 Express、Sequelize、Redis 这些词。先把后端当成一件很朴素的事：

- 前端发一个请求
- 后端收到这个请求
- 后端去数据库拿数据
- 后端把结果再返回给前端

这个项目的后端，本质上就是在做这件事。

## 1. 先理解：后端到底是什么

你现在这个项目分成三块：

- 前端：页面长什么样、按钮怎么点、图片怎么显示
- 后端：前端要数据时，去哪里拿、怎么处理、怎么返回
- 数据库：真正存数据的地方

比如首页想显示“分类列表”：

1. 前端请求 `/api/classify`
2. 后端收到这个请求
3. 后端去数据库里的 `categories` 表查数据
4. 后端把分类 JSON 返回给前端
5. 前端把这些数据渲染成页面

所以你看后端代码时，最核心的问题永远是：

- 这个接口是谁接住的
- 它去哪拿数据
- 它最后返回了什么

## 3. 一个请求到底怎么走

这是你最应该先记住的东西。

当前项目里，一个典型请求会这样走：

```text
前端请求 /api/classify
-> app.js
-> accessKey 中间件
-> routes/category.js
-> controllers/categoryController.js
-> models/Category.js
-> MySQL
-> response.js
-> 返回给前端
```

你可以先不用背所有文件名，但要记住这个顺序：

1. 入口接请求
2. 中间件先处理
3. 路由决定给谁
4. controller 写业务
5. model 查数据库
6. 返回结果

只要这个主线不丢，你后面看任何接口都不会太乱。

## 5. app.js 要怎么看

先看：

- [app.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/app.js)

这个文件你不用逐行抠，先看三件事：

1. 有没有挂中间件
2. 有没有挂路由
3. 最后有没有启动服务

你会看到它做了这些事：

- 开启跨域 `cors`
- 解析 JSON 请求体
- 挂日志中间件
- 所有 `/api` 先走 `accessKey` 校验
- 注册分类、轮播图、壁纸、公告、用户路由
- 启动服务端口

读这个文件时，你脑子里只要形成一句话就够了：

“这个文件负责把整个后端组装起来。”

## 6. 路由文件要怎么看

比如看：

- [category.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/routes/category.js)

你会看到类似：

```js
router.get('/classify', categoryController.getClassify)
```

这句话可以直接翻译成：

- 当前端请求 `GET /classify`
- 就交给 `categoryController.getClassify` 去处理

所以 `routes` 层的作用其实很简单：

- 它自己不做复杂业务
- 它只是“分发请求”

你以后看每个接口，第一步都可以先去 `routes` 找。

## 7. Controller 要怎么看

Controller 是后端最重要的地方。

比如：

- [categoryController.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/controllers/categoryController.js)

你可以这样看它：

### 第一步：看它接收了什么参数

比如：

- `limit`
- `skip`
- `select`

这说明这个接口支持分页，也支持“只拿精选分类”。

### 第二步：看它有没有做校验或兼容

这里能看到：

- 分类接口只认 `limit + skip`
- `select=true` 时只拿精选分类

### 第三步：看它有没有查缓存

这里会先读 Redis。

意思是：

- 如果之前查过，就直接返回缓存
- 不用每次都查数据库

### 第四步：看它有没有查数据库

这里会调用 `Category.findAndCountAll(...)`

这一步你可以简单理解成：

- 去数据库查 `categories` 表
- 顺便把总数也拿回来

### 第五步：看它最后返回什么

最后会调用 `success(...)`

也就是把数据包装成统一的 JSON 返回给前端。

你以后看任何 controller，都按这 5 步看，基本都能拆开。

## 8. Model 要怎么看

比如：

- [Category.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/models/Category.js)

你可以把它当成：

- “数据库表 `categories` 的代码说明书”

比如这里会定义：

- `name`
- `picurl`
- `sort`
- `select`
- `wallpaper_count`

这就对应数据库表里的列。

所以 `models` 的阅读重点是：

- 这个表有哪些字段
- 每个字段是干什么的
- 有没有默认值
- 表名是什么
- 时间字段是不是自动维护

你现在最值得特别记住的是：

- [Wallpaper.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/models/Wallpaper.js)

意思是：

- 代码里叫 `smallPicurl`
- 数据库真实列名叫 `small_picurl`

如果以后你看到相关报错，这里就是重点检查点。

## 9. 中间件要怎么看

你先重点看这两个：

- [accessKey.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/middlewares/accessKey.js)
- [logger.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/middlewares/logger.js)

### accessKey.js

它的作用很简单：

- 请求来了先看有没有 `access-key`
- 没有或者不对，就直接返回错误

所以你之前在 Apifox 里一直报：

```json
{
  "errCode": 401,
  "errMsg": "无效的 access-key"
}
```

根因就是这里在拦。

### logger.js

它会记录：

- 请求时间
- 请求方法
- 请求路径
- 状态码
- 耗时

你可以理解成后端的“访问日志”。

## 10. response.js 为什么重要

看这里：

- [response.js](C:/Users/hulian/Desktop/huliang/bizi/univue3/backend-code/src/utils/response.js)

这个文件的作用是：

- 让所有接口返回格式尽量统一

比如成功大概长这样：

```json
{
  "errCode": 0,
  "errMsg": "查询成功",
  "data": [],
  "timeCost": 0
}
```

失败大概长这样：

```json
{
  "errCode": 400,
  "errMsg": "参数不完整",
  "data": null,
  "timeCost": 0
}
```

所以你以后如果想改“统一返回格式”，就应该先看这个文件，而不是到每个 controller 里乱改。

## 12. 当前项目里你最容易踩的坑

你读代码时，最好重点记住这些坑：

- 所有接口都要 `access-key`
- 返回里同时保留 `id` 和 `_id`
- 新参数推荐用：
  - `wallpaperId`
  - `score`
  - `limit`
  - `skip`
- 旧参数仍兼容：
  - `wallId`
  - `userScore`
- `userInfo` 不传 `userId` 时会回退到第一个用户
- 历史兼容路径仍然存在：
  - `/homeBanner`
  - `/wallNewsList`
  - `/wallNewsDetail/:id`

也就是说，这个后端现在不只是“新代码”，还是一套带兼容层的代码。

## 13. 如果你想真正开始学后端，建议怎么练

最适合你的方式不是一下子看完整个项目，而是这样练：

1. 先盯住一个接口
2. 从 `routes` 找到它
3. 进 `controller` 看逻辑
4. 再去 `model` 看字段
5. 最后在 Apifox 里调一下

比如你今天只练：

- `/api/classify`

明天再练：

- `/api/detailWall/:id`

后天再练：

- `/api/setupScore`

这样你会比“把所有文件都打开”学得快很多。

## 14. 你现在不需要一次学会的东西

这些东西你现在知道名字就够了，不用马上全懂：

- `Sequelize` 的高级查询
- Redis 缓存策略优化
- PM2 进程管理细节
- Nginx 反向代理细节
- MySQL 索引设计

你现在真正需要先掌握的是：

- 请求怎么进来
- 路由怎么分发
- controller 怎么拿参数和返回数据
- model 怎么对应数据库表

把这四件事读顺，后端就不会再那么陌生。
