# 项目概览

## 1. 项目组成

这是一个壁纸项目，当前仓库同时包含：

- uni-app 前端
- Node.js + Express 后端
- 文档与部署脚本

## 2. 技术栈

### 前端

- uni-app
- Vue 3
- HBuilderX 运行与打包

### 后端

- Node.js
- Express
- Sequelize
- MySQL
- Redis
- PM2
- Nginx

## 3. 当前目录结构

```text
univue3/
├── api/                    # 前端接口定义
├── common/                 # 公共资源
├── components/             # 前端组件
├── config/                 # 前端环境配置
├── docs/                   # 文档
├── pages/                  # 页面
├── static/                 # 静态资源
├── utils/                  # 前端工具
├── backend-code/           # 后端工程
├── deploy-h5-local.sh      # H5 本地上传脚本
├── deploy-h5-server.sh     # H5 服务器配置脚本
└── setup-config.sh         # 前端配置向导
```

## 4. 后端接口现状

后端当前实际注册的路由来源于：

- `backend-code/src/routes/category.js`
- `backend-code/src/routes/banner.js`
- `backend-code/src/routes/wallpaper.js`
- `backend-code/src/routes/notice.js`
- `backend-code/src/routes/user.js`

## 5. 当前最重要的已知问题

### 前后端接口命名未完全对齐

前端 `api/apis.js` 中使用的这组历史路径：

- `/homeBanner`
- `/wallNewsList`
- `/wallNewsDetail/:id`

现在已经在后端补了兼容路由，对应可正常映射到：

- `/banner`
- `/notice`
- `/notice/:id`

这意味着当前联调可以先保持前端代码不动，同时后端也保留了更简洁的新路由，后续如果要统一命名，可以再做一次前端收口。

## 6. 运行方式

### 前端

- 不依赖根目录 `package.json`
- 通常通过 HBuilderX 打开项目并运行

### 后端

```bash
cd backend-code
npm install
npm run dev
```
