# 前后端配置

这份文档关注三个问题：前端请求打到哪里、请求头里的 `access-key` 从哪里来、不同环境应该怎么配。

## 1. 相关文件

- `config/env.example.js`：前端配置模板
- `config/env.js`：前端实际使用的配置
- `utils/request.js`：统一请求封装
- `api/apis.js`：前端接口定义
- `backend-code/.env.example`：后端环境变量模板
- `backend-code/.env`：后端实际运行配置

## 2. 前端配置方式

前端通过 `config/env.js` 导出两个核心值：

- `BASE_URL`：请求基础地址
- `ACCESS_KEY`：默认请求密钥

当前请求流程：

1. `api/apis.js` 定义接口路径
2. `utils/request.js` 将路径拼接到 `BASE_URL`
3. `getAccessKey()` 返回请求头中的 `access-key`

## 3. 常见前端环境示例

### 本机联调

```javascript
development: {
  baseURL: 'http://localhost:3000/api',
  accessKey: 'key123456'
}
```

### 局域网真机调试

```javascript
development: {
  baseURL: 'http://192.168.1.100:3000/api',
  accessKey: 'key123456'
}
```

### 生产环境

```javascript
production: {
  baseURL: 'https://api.yourdomain.com/api',
  accessKey: 'your_production_key'
}
```

## 4. 动态 Access Key

`config/env.js` 中的 `getAccessKey()` 支持从页面参数读取 `accessKey`。如果页面 URL 中携带了 `accessKey`，它会覆盖默认配置。

示例：

```javascript
uni.navigateTo({
  url: '/pages/index/index?accessKey=custom_key'
});
```

## 5. 后端环境变量

`backend-code/.env` 至少需要配置这些字段：

```env
NODE_ENV=production
PORT=3000

DB_HOST=localhost
DB_PORT=3306
DB_NAME=wallpaper
DB_USER=wallpaper_user
DB_PASSWORD=your_password

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password

ACCESS_KEYS=key123456,key789012
```

说明：

- `ACCESS_KEYS` 支持多个值，用逗号分隔。
- 生产环境不要直接使用模板中的示例密码。

## 6. 联调检查清单

- `config/env.js` 的 `baseURL` 是否指向当前 API
- `config/env.js` 的 `accessKey` 是否包含在后端 `ACCESS_KEYS` 中
- 后端是否已成功启动在 `PORT`
- 浏览器 Network 中请求头是否包含 `access-key`

## 7. 当前实现的一个已知差异

前端 `api/apis.js` 里仍保留了历史接口名：

- `/homeBanner`
- `/wallNewsList`
- `/wallNewsDetail/:id`

但后端 `backend-code/src/routes/` 当前实际暴露的是：

- `/banner`
- `/notice`

并且目前没有实现公告详情接口。文档已经按“当前代码事实”整理；如果后续修复这组接口，请同步更新本文件和 [API 测试](../reference/API测试.md)。
